Functions

Plugin.validate_opts (opts)
Validates the plugin options.

This function validates the given options table against a schema and returns the validated options. If the options are not valid, an error is thrown.

Parameters:

  • opts (table): The options table to validate.

Returns:

    table: The validated options table if the options are valid, otherwise nil.
Plugin.apply_user_options (user_opts)
Applies the user options to the default options.

This function replaces the user options with the default options and returns the resulting options table.

Parameters:

  • user_opts (table): The user options to apply.

Returns:

    table: The resulting options table.
Plugin.setup (user_opts)
Sets up the plugin with the user options.

This function sets up the buffer-closer plugin with the given options.

Parameters:

  • user_opts (table): The user options to apply to the plugin, it will replace the default options.

See also:

    buffer-closer.modules.timer.init

Tables

DEFAULT_OPTIONS
Plugin module provides options for the `buffer-closer` plugin. Default options for the `buffer-closer` plugin.

Fields:

  • min_remaining_buffers The minimum number of buffers to keep open.
  • retirement_minutes The number of minutes after which a buffer is considered retired.
  • checking_interval_minutes The interval, in minutes, at which to check for retired buffers.
  • run_when_min_buffers_reached A table of options for running the buffer-closer when the number of buffers is greater than the given number.
    • enabled Whether to run the buffer-closer when the number of buffers is greater than the given number.
    • min_buffers The number of buffers to reach before running the buffer-closer.
  • excluded A table of excluded filetypes, buffer types, and filenames.
    • filetypes A table of excluded filetypes.
    • buftypes A table of excluded buffer types.
    • filenames A table of excluded filenames.
  • ignore_working_windows Whether to ignore buffers open in windows.
